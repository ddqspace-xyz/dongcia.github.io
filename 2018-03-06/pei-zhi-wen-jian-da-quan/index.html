配置文件大全

发表于 2018-03-06

开发中会碰到各式各样的配置文件，语法、使用场景各有不同，故简单汇总一下。

1 .ini

多见于windows系统下，是windows 下采用的配置文件格式，.ini 文件是Initialization File的缩写。典型的如：windows安装mysql的配置文件my.ini。

语法

  • ini文件由section、key和value组成
  • section用中括号，key、value对用之间用=

解析器

python

使用ConfigParser

java

使用ini4j

2 .yml

YAML 是专门用来写配置文件的语言，参考了XML、Python等语言，非常简洁和强大，目前在前端开源框架中经常见到。

语法

  • 大小写敏感
  • 使用缩进表示层级关系
  • 缩进时不允许使用Tab键，只允许使用空格。
  • 缩进的空格数目不重要，只要相同层级的元素左侧对齐即可

解析器

python

java

3 .xml

广泛使用的一种标记语言，比如maven配置pom.xml等。

语法

  • 与HTML相似，使用<>开始，</>结束
  • 注释： <!---->

解析器

对xml的解析有几种方法，SAX、DOM和ElementTree，将XML数据解析成一棵DOM树或ElementTree比较容易理解，xml的结构就像树的结构。

java

使用dom4j

4 .json

json作为配置文件非常常见，语法格式也是简单明了，各大语言都有专门解析json的工具，如：java的Gson、fastjson等，python标准库也带有json库。

5 .properties

.properties的配置文件在java中常用，可以说是写java配置文件的标配。

语法

  • key=value，等号左右不能空格
  • key 可以用.做分割以表示类似ini文件里section的概念

解析器

java

总结

就易用性和可读性而言，yaml算是最友好的一种配置文件相比json更为简单，当然json也不错；xml是比较晦涩的一种，给人感觉冗长而复杂，因历史原因xml还是广泛存在的。此外ini在windows上广泛使用，在linux则比较少见，properties则是java项目比较常用的了。
